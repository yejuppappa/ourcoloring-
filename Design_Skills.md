# Design_Skills.md — OurColoring 디자인 스킬 가이드

> Claude Code가 UI/UX 구현 시 참고하는 디자인 원칙과 실행 가이드.
> 이 문서는 방향을 제시하며, 세부 구현은 Claude Code가 자율 판단한다.

---

## 🎨 디자인 철학

### 이 서비스의 사용자를 기억하라
- **부모**가 **아이와 함께** 사용하는 서비스다.
- 신뢰, 따뜻함, 재미가 공존해야 한다.
- "테크 스타트업" 느낌이 아니라 "따뜻한 문구점" 느낌에 가깝다.
- 갤러리는 "잘 정리된 색칠공부 서점" 느낌 — 탐색이 즐거워야 한다.
- **"도구 사이트"가 아니라 "플랫폼"으로 보여야 한다** — 사이드바 네비게이션으로 Freepik 같은 구조감을 준다.

### 피해야 할 것
- 차갑고 기업적인 느낌
- AI 생성물 특유의 뻔한 그라데이션 (보라-파랑 등)
- 과도한 장식이나 애니메이션 (부모는 빠르게 찾고 싶다)
- 콘텐츠가 비어 보이는 허전한 레이아웃

## 🖋️ 타이포그래피

- 한국어: 부드럽고 읽기 쉬운 산세리프 (Pretendard, Noto Sans KR 등)
- 영어: 둥글고 따뜻한 느낌
- Google Fonts 사용, font-display: swap
- 최종 선택은 Claude Code 자율

---

## 🎨 컬러 시스템

- **배경:** 따뜻한 크림/아이보리 (순백색 #FFF 피함)
- **텍스트:** 부드러운 다크 (순흑색 #000 피함)
- **액센트(CTA):** 채도 높은 따뜻한 색 하나
- **보조:** 파스텔 2~3색
- **카테고리 컬러:** A(놀이)/B(교육)/C(창의) 각각 구분 가능한 색상 계열
- WCAG AA 명도 대비 4.5:1 이상

## 📱 반응형 디자인 (3단계)

### 원칙
- 모바일 퍼스트로 설계, 태블릿/PC로 확장
- 같은 콘텐츠, 다른 레이아웃 — 콘텐츠 숨기기/추가 아님, 배치만 변경

### 브레이크포인트
- **모바일:** ~ 767px (기본)
- **태블릿:** 768px ~ 1023px
- **PC:** 1024px ~
- 세부 브레이크포인트 추가는 Claude Code 자율

### 디바이스별 고려사항

**모바일 (~ 767px)**
- 터치 타겟 최소 44x44px
- **사이드바 숨김** → 햄버거 메뉴 또는 슬라이드 아웃으로 접근
- 갤러리: 2열 그리드
- 변환 도구: 세로 스택 (업로드 → 옵션 → 결과)

**태블릿 (768px ~ 1023px)**
- 아이가 직접 탐색할 수 있는 큰 썸네일
- **사이드바 축소** (아이콘만 표시) 또는 토글 가능
- 갤러리: 3~4열 그리드
- 변환 도구: 여유 있는 레이아웃 (좌우 배치 가능)
- 카테고리 탐색이 편해야 함

**PC (1024px ~)**
- **사이드바 항상 표시** — 좌측에 고정, 메인 콘텐츠는 오른쪽
- 갤러리: 4~6열 와이드 그리드
- 변환 도구: 좌(업로드+옵션) / 우(결과) 배치
- 충분한 여백으로 여유 있는 느낌

## 🏠 랜딩 페이지 (메인)

### 레이아웃
- PC: 사이드바(좌) + 메인 콘텐츠(우) 구조
- 메인 콘텐츠 영역에 아래 구성 요소 배치

### 구성 요소
1. **히어로 섹션:** 서비스를 한 눈에 이해할 수 있는 비주얼 + 카피
   - "색칠하면서 놀고, 배우고, 만든다" 방향의 메시지
   - 갤러리 탐색 CTA + 사진 변환 CTA 양쪽 다 노출
2. **갤러리 하이라이트:** A/B/C 카테고리 맛보기 (각 3~4개 도안 썸네일)
3. **사진→도안 변환 도구:** 기존 업로드+변환 영역
4. **사용법 안내:** 3단계 (탐색/업로드 → 선택/변환 → 프린트)
5. **CTA:** "갤러리 탐색하기" + "내 사진 변환하기"

### 제약조건
- 갤러리가 비어있는 초기에도 어색하지 않아야 함 (콘텐츠 추가 예정 표시 또는 유연한 그리드)
- **사이드바에서 이미 카테고리 접근이 가능하므로**, 랜딩 페이지의 갤러리 하이라이트는 "맛보기" 역할
- 배치/순서/디자인은 Claude Code 자율

## 📚 갤러리 UI

### 갤러리 메인 (/gallery/)
- **What:** 놀이(A) / 교육(B) / 창의(C) 세 카테고리를 한눈에 보여주는 페이지
- **Why:** 사용자가 원하는 유형의 도안을 빠르게 찾을 수 있어야 함
- **제약조건:**
  - 3개 카테고리가 시각적으로 명확히 구분 (아이콘, 색상, 레이블)
  - 각 카테고리에서 인기/최신 도안 미리보기
  - 카테고리 클릭 시 해당 서브페이지로 이동
  - 디자인은 Claude Code 자율

### 카테고리 페이지 (예: /gallery/play/)
- **What:** 해당 카테고리의 서브카테고리 목록 + 도안 그리드
- **Why:** 서브카테고리로 세분화하여 원하는 도안을 쉽게 찾게
- **제약조건:**
  - 서브카테고리 필터/탭 제공 (공룡, 자동차, 동물 등)
  - 도안 그리드: 썸네일 + 이름 + 난이도 표시
  - 난이도/연령별 필터 가능
  - 디자인은 Claude Code 자율

### 개별 도안 페이지
- **What:** 도안 하나의 상세 페이지
- **Why:** SEO (개별 URL이 검색 노출), 다운로드/공유의 명확한 지점
- **제약조건:**
  - 도안 이미지 크게 표시
  - 다운로드 버튼 (PNG/PDF) 명확하게
  - 난이도, 타겟 연령, 카테고리 표시
  - "이 도안도 좋아할 수 있어요" — 관련 도안 추천
  - 소셜 공유 버튼
  - "OurColoring" 워터마크는 이미지에 포함 (다운로드 파일에)
  - 디자인은 Claude Code 자율

## 🤖 AI 변환 UX (기존 유지)

### 배경 모드 선택 UI
- **3가지 모드:**
  - 🖼️ **배경 유지** — "원래 배경과 함께"
  - ✂️ **배경 제거** — "주인공만 깔끔하게"
  - 🎨 **배경 창작** — "AI가 어울리는 배경을 새로 그려요"
- **제약조건:**
  - 3개 모드는 시각적으로 명확히 구분
  - 기본 선택값 없음 — 사용자가 반드시 선택해야 변환 가능
  - 배치/디자인은 Claude Code 자율

### 난이도 선택 UI
- **3단계:**

| 난이도 | 라벨 제안 | 설명 |
|--------|----------|------|
| **상 (기본값)** | "디테일" 또는 "정교하게" | 실물과 가장 비슷한 정교한 도안 |
| **중** | "보통" 또는 "적당하게" | 7~9세에 적합한 중간 단순화 |
| **하** | "쉬움" 또는 "심플하게" | 4~6세 유아도 색칠 가능 |

- **제약조건:**
  - 기본값은 **상 (디테일)**
  - 배치/디자인은 Claude Code 자율

### AI 로딩 UX
- "AI가 도안을 그리는 중..." 같은 친근한 메시지
- 진행감을 주는 애니메이션
- 구체적인 디자인은 Claude Code 자율

### 변환 실패 처리
- "앗, 변환에 실패했어요. 다시 시도해볼까요?"
- 재시도 버튼 제공
- 일일 무료 횟수는 실패 시 차감하지 않음

## 🧭 네비게이션 (사이드바 기반)

### What
Freepik처럼 **좌측 사이드바**를 중심으로 한 플랫폼형 네비게이션. 갤러리와 사진 변환이 동등한 양대 축으로 보여야 한다.

### Why
- 기존 상단 헤더 네비게이션만으로는 "도구 사이트"처럼 보임
- 사이드바가 있으면 "플랫폼"처럼 보이고, 카테고리 탐색이 자연스러움
- 부모가 아이와 함께 카테고리를 훑어보며 도안을 고르는 경험에 최적

### 사이드바 구조
```
🏠 홈
🎨 놀이 도안
   공룡 / 자동차 / 동물 / 유니콘 / 직업 / 계절
📚 교육 도안
   알파벳 / 숫자 / Color by Number / 세계문화 / 이름
🧠 창의 도안
   다음장면 / 반쪽완성 / 캐릭터디자인 / 나만의공룡
📷 사진→도안 변환
🖨️ 프린트 가이드
```

### 제약조건
- **PC (1024px ~):** 사이드바 항상 표시. 메인 콘텐츠는 사이드바 오른쪽에 배치.
- **태블릿 (768px ~ 1023px):** 사이드바 접힘 가능 (아이콘만 표시 또는 토글)
- **모바일 (~ 767px):** 사이드바 숨김 → 햄버거 메뉴 또는 슬라이드 아웃
- **언어 전환:** 사이드바 하단에 한국어 ↔ 영어 전환
- **현재 위치:** 활성 메뉴 항목 하이라이트
- **갤러리와 사진변환이 동등하게 보여야 함** — 사진변환이 메인이고 갤러리가 부속이 아님
- 사이드바 디자인, 아이콘, 색상, 애니메이션: Claude Code 자율

## 🧩 컴포넌트 디자인 가이드

### 업로드 영역
- 서비스의 핵심 CTA 중 하나
- 비어 있을 때: 점선 테두리 + 아이콘 + "사진을 올려보세요"
- 모바일: 탭하면 파일 선택 + 카메라 옵션

### 도안 카드 (갤러리 그리드용)
- 썸네일 이미지 + 도안 이름 + 난이도 표시
- 호버/탭 시 미리보기 또는 상세 페이지 이동
- 깔끔하고 일관된 카드 디자인
- 구체적 디자인은 Claude Code 자율

### 카테고리 카드
- A(놀이)/B(교육)/C(창의)를 대표하는 비주얼
- 각 카테고리의 성격이 직관적으로 드러나야 함
- 아이콘 + 짧은 설명 + 도안 수 표시

### 변환 옵션 영역
- 배경 모드 3개 + 난이도 3단계를 직관적으로 배치
- 모바일에서 한 화면에 보일 수 있도록 컴팩트하게
- "변환하기" CTA 버튼

### 다운로드 버튼
- 눈에 잘 띄어야 함 (CTA 컬러)
- PNG / PDF 선택 직관적
- 갤러리 도안과 AI 변환 결과 모두 동일한 다운로드 경험

### 일일 제한 카운터
- "오늘 남은 변환: 2/3" 간결하게
- AI 변환에만 해당, 갤러리 다운로드는 무제한
- 제한 도달 시 부드러운 안내

### 푸터 (Footer)
- **What:** 모든 페이지 하단에 표시되는 공통 영역
- **Why:** AdSense 승인 필수 요건. 사이트 신뢰도와 법적 요건 충족.
- **필수 포함 링크:** 개인정보처리방침, 이용약관, 문의하기, 소개
- **추가:** © OurColoring, 언어 전환 링크
- **제약조건:** 모든 페이지에 일관되게 표시, 디자인은 Claude Code 자율

- 여백: 상하좌우 15~20mm
- 선화는 흑백으로 선명하게

## 📱 성능 가이드

- **Lighthouse 90+ 목표**
- 이미지 최적화: WebP 우선, lazy loading (갤러리 그리드 필수)
- 폰트: 서브셋 + font-display: swap
- JS: React Island 패턴으로 필요한 곳만 hydrate
- Core Web Vitals: LCP < 2.5s, FID < 100ms, CLS < 0.1
- 갤러리 이미지 많아질수록 성능 관리 중요 — 무한 스크롤 또는 페이지네이션 고려
