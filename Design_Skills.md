# Design_Skills.md — OurColoring 디자인 스킬 가이드

> Claude Code가 UI/UX 구현 시 참고하는 디자인 원칙과 실행 가이드.
> 이 문서는 방향을 제시하며, 세부 구현은 Claude Code가 자율 판단한다.

---

## 🎨 디자인 철학

### 이 서비스의 사용자를 기억하라
- **부모**가 **아이 사진**을 올리는 서비스다.
- 신뢰, 따뜻함, 재미가 공존해야 한다.
- "테크 스타트업" 느낌이 아니라 "따뜻한 문구점" 느낌에 가깝다.

### 피해야 할 것
- 차갑고 기업적인 느낌
- AI 생성물 특유의 뻔한 그라데이션 (보라-파랑 그라데이션 등)
- 과도한 장식이나 애니메이션 (부모는 빠르게 변환하고 싶다)
- Generic 시스템 폰트 (Inter, Roboto, Arial)

## 🖋️ 타이포그래피

### 원칙
- 한국어 서체는 부드럽고 읽기 쉬운 것을 우선한다.
- 영어 서체는 둥글고 따뜻한 느낌을 준다.
- 제목과 본문의 서체 대비로 시각적 리듬을 만든다.

### 추천 방향 (최종 선택은 Claude Code 자율)
- **한국어 본문:** Pretendard, Noto Sans KR, Wanted Sans 등 가독성 높은 산세리프
- **영어 제목:** 둥글고 개성 있는 서체 (Quicksand, Nunito, Baloo 2, Comfortaa 등에서 영감)
- **크기 체계:** 모바일 기준 본문 16px, 제목은 계층적으로

### 주의
- Google Fonts에서 사용 가능한 서체만 쓴다 (셀프 호스팅 또는 CDN).
- 서체 로딩이 성능에 미치는 영향을 고려한다 (font-display: swap).

## 🎨 컬러 시스템

### 방향
- **밝고 따뜻한** 기본 톤
- 아이들의 색칠공부를 연상시키는 **부드러운 파스텔** 기반
- 단, 파스텔만으로는 밋밋하므로 **생동감 있는 액센트** 하나를 강하게

### 컬러 구성 원칙
- **배경:** 따뜻한 크림/아이보리 계열 (순백색 #FFF 피함)
- **주 텍스트:** 부드러운 다크 (순흑색 #000 피함)
- **액센트(CTA):** 채도 높은 따뜻한 색 하나 (변환 버튼, 핵심 액션)
- **보조 컬러:** 파스텔 2~3색 (섹션 구분, 카드 배경 등)
- CSS 변수로 일관성 있게 관리

### 주의
- 접근성: WCAG AA 기준 명도 대비 4.5:1 이상 (텍스트)
- CTA 버튼은 배경과 충분한 대비를 가져야 한다
- 다크모드는 MVP에서 제외

- **제약조건:** 모바일에서 핵심 액션(업로드, 다운로드)에 쉽게 접근 가능해야 한다.
- 4단계의 배치 방식, 순서, 전환 방식은 Claude Code가 자율 판단한다.

## ✨ 인터랙션 & 모션

### 원칙
- 목적 있는 애니메이션만 사용한다.
- "화려함"보다 "명확한 피드백"이 우선이다.
- 부모는 바쁘다. 기다리게 하지 마라.

### 적용 포인트
- **업로드 시:** 드래그 오버 → 영역 하이라이트 / 프로그레스 표시
- **변환 중:** 로딩 인디케이터 (재미있되 과하지 않게)
- **변환 완료:** 결과 페이드인 / 원본→결과 전환 효과
- **버튼 호버/클릭:** 미세한 스케일 + 색상 변화
- **페이지 로드:** 부드러운 스태거드 페이드인 (랜딩 페이지)

### 주의
- 변환 처리 자체가 3초 이내여야 하므로, 로딩 애니메이션이 과하면 안 된다.
- `prefers-reduced-motion` 미디어 쿼리 존중.
- CSS 애니메이션 우선. JS 애니메이션은 필요할 때만.

## 🧩 컴포넌트 디자인 가이드

### 업로드 영역
- 서비스의 첫인상이자 핵심 CTA
- 비어 있을 때: 점선 테두리 + 아이콘 + "사진을 올려보세요" 텍스트
- 호버/드래그 시: 배경색 변화, 테두리 강조
- 모바일: 탭하면 파일 선택 + 카메라 옵션

### 변환 결과 뷰
- 원본과 결과를 쉽게 비교할 수 있으면 좋다 (슬라이더 비교 또는 탭 전환)
- 결과 이미지 주변에 "색칠공부 용지" 같은 프레임 느낌을 주면 감성적

### 다운로드 버튼
- 눈에 잘 띄어야 한다 (CTA 컬러)
- PNG / PDF 선택이 직관적이어야 한다
- 다운로드 완료 피드백

### 일일 제한 카운터
- 압박감 없이 자연스럽게 표시
- "오늘 남은 변환: 2/3" 같은 간결한 형태
- 제한 도달 시 부드러운 프리미엄 안내 (공격적 팝업 ✗)

### 랜딩 페이지
- 히어로: 한 눈에 서비스를 이해할 수 있는 비주얼 + 카피
- 사용법: 3단계 (업로드 → 변환 → 프린트) 시각적 설명
- 샘플 결과: 변환 전후 비교 이미지
- 사회적 증거 (추후): 사용자 후기, 변환 수 카운터
- CTA: "지금 바로 시작하기" → 업로드 영역으로 스크롤

## 🌐 다국어 디자인 고려

- 한국어는 영어보다 같은 내용이 짧은 경우가 많다. 레이아웃이 양쪽에서 잘 작동해야 한다.
- 버튼 텍스트 길이 변화에 대비 (auto width 또는 min-width)
- 날짜/시간 포맷 로컬라이제이션 (일일 제한 리셋 시간 등)

## 🖨️ 프린트 최적화

### PDF 출력 가이드
- A4 (210mm × 297mm) 기준
- 여백: 상하좌우 15~20mm
- 선화는 흑백으로 선명하게 (그레이 톤 피함)
- 선 굵기: 프린트 시 잘 보이도록 최소 1.5~2px (화면 기준)
- 하단에 작은 watermark (ourcoloring.com) — 프리미엄은 제거

## 📱 성능 가이드

- **Lighthouse 90+ 목표** (Performance, Accessibility, Best Practices, SEO)
- 이미지 최적화: WebP 우선, lazy loading
- 폰트: 서브셋 + font-display: swap
- CSS: Tailwind purge로 미사용 클래스 제거
- JS: React Island 패턴으로 필요한 곳만 hydrate
- Core Web Vitals: LCP < 2.5s, FID < 100ms, CLS < 0.1

## 🔒 신뢰 & 프라이버시 시그널

부모가 아이 사진을 올리는 서비스이므로, 신뢰 시그널이 중요하다:
- "사진은 서버에 저장되지 않습니다" — 업로드 영역 근처에 명시
- "모든 처리는 브라우저에서 이루어집니다" — 기술적 안전 강조
- 프라이버시 정책 페이지 (간결하게)
- HTTPS 기본 (Cloudflare 자동)
