# Design_Skills.md — OurColoring 디자인 스킬 가이드

> Claude Code가 UI/UX 구현 시 참고하는 디자인 원칙과 실행 가이드.
> 이 문서는 방향을 제시하며, 세부 구현은 Claude Code가 자율 판단한다.

---

## 🎨 디자인 철학

### 이 서비스의 사용자를 기억하라
- **부모**가 **아이 사진**을 올리는 서비스다.
- 신뢰, 따뜻함, 재미가 공존해야 한다.
- "테크 스타트업" 느낌이 아니라 "따뜻한 문구점" 느낌에 가깝다.

### 피해야 할 것
- 차갑고 기업적인 느낌
- AI 생성물 특유의 뻔한 그라데이션 (보라-파랑 등)
- 과도한 장식이나 애니메이션 (부모는 빠르게 변환하고 싶다)

---

## 🖋️ 타이포그래피

- 한국어: 부드럽고 읽기 쉬운 산세리프 (Pretendard, Noto Sans KR 등)
- 영어: 둥글고 따뜻한 느낌
- Google Fonts 사용, font-display: swap
- 최종 선택은 Claude Code 자율

---

## 🎨 컬러 시스템

- **배경:** 따뜻한 크림/아이보리 (순백색 #FFF 피함)
- **텍스트:** 부드러운 다크 (순흑색 #000 피함)
- **액센트(CTA):** 채도 높은 따뜻한 색 하나
- **보조:** 파스텔 2~3색
- WCAG AA 명도 대비 4.5:1 이상

---

## 📐 레이아웃 & 공간

### 모바일 퍼스트
- 모든 컴포넌트는 모바일(360px~)에서 먼저 디자인
- 브레이크포인트: sm(640px), md(768px), lg(1024px)
- 터치 타겟 최소 44x44px

### 핵심 플로우
**업로드 → 배경 모드 선택 → 난이도 선택 → AI 변환(로딩) → 결과 확인 → 다운로드**

6단계를 한 페이지 안에서 완결. 배치 방식은 Claude Code 자율.

## 🤖 AI 변환 UX

### 배경 모드 선택 UI
- **What:** 사용자가 3가지 배경 모드 중 하나를 선택
- **Why:** 각 모드의 결과물이 크게 다르므로 사전 선택 필요
- **3가지 모드:**
  - 🖼️ **배경 유지** — "원래 배경과 함께"
  - ✂️ **배경 제거** — "주인공만 깔끔하게"
  - 🎨 **배경 창작** — "AI가 어울리는 배경을 새로 그려요"
- **제약조건:**
  - 3개 모드는 시각적으로 명확히 구분 (카드형, 아이콘 등)
  - 기본 선택값 없음 — 사용자가 반드시 선택해야 변환 가능
  - "배경 창작" 모드에 작은 안내: "AI가 배경을 상상해서 그려요"
  - 배치/디자인은 Claude Code 자율

### 난이도 선택 UI
- **What:** 사용자가 색칠할 아이의 연령에 맞는 난이도를 선택
- **Why:** 어린 아이(4~6세)에게 복잡한 도안은 색칠이 어렵다. 연령에 맞는 난이도를 제공해야 만족도가 높아진다.
- **3단계:**

| 난이도 | 라벨 제안 | 설명 | 시각적 표현 |
|--------|----------|------|-----------|
| **상 (기본값)** | "디테일" 또는 "정교하게" | 실물과 가장 비슷한 정교한 도안 | 복잡한 선화 아이콘 |
| **중** | "보통" 또는 "적당하게" | 7~9세에 적합한 중간 단순화 | 중간 선화 아이콘 |
| **하** | "쉬움" 또는 "심플하게" | 4~6세 유아도 색칠 가능한 큰 윤곽선 | 단순한 선화 아이콘 |

- **제약조건:**
  - 기본값은 **상 (디테일)** — 실물 유사도가 핵심 가치이므로
  - 난이도 설명 페이지 또는 툴팁으로 각 단계의 차이를 보여주면 좋음
  - 난이도별 샘플 이미지를 보여주면 이해가 빠름 (같은 원본 → 상/중/하 결과 비교)
  - 배치/디자인은 Claude Code 자율

### AI 로딩 UX
- **What:** AI가 도안을 생성하는 동안 사용자에게 보여줄 로딩 상태
- **Why:** Grok은 빠르지만 여전히 수 초 대기 시간이 있다. 기다림을 참을 만하게 만들어야 한다.
- **제약조건:**
  - "AI가 도안을 그리는 중..." 같은 친근한 메시지
  - 진행감을 주는 애니메이션
  - 과하지 않되 지루하지 않게
  - 구체적인 디자인은 Claude Code 자율

### 변환 실패 처리
- **What:** API 에러, 타임아웃 등 실패 시 안내
- **제약조건:**
  - "앗, 변환에 실패했어요. 다시 시도해볼까요?"
  - 재시도 버튼 제공
  - 일일 무료 횟수는 실패 시 차감하지 않음

## 🧩 컴포넌트 디자인 가이드

### 업로드 영역
- 서비스의 첫인상이자 핵심 CTA
- 비어 있을 때: 점선 테두리 + 아이콘 + "사진을 올려보세요"
- 모바일: 탭하면 파일 선택 + 카메라 옵션

### 변환 옵션 영역 (배경 모드 + 난이도)
- 사진 업로드 후 표시
- 배경 모드 3개 + 난이도 3단계를 직관적으로 배치
- 모바일에서 한 화면에 보일 수 있도록 컴팩트하게
- "변환하기" CTA 버튼

### 변환 결과 뷰
- 원본과 결과를 쉽게 비교 가능하면 좋음
- 결과 이미지 주변에 "색칠공부 용지" 같은 프레임 느낌

### 다운로드 버튼
- 눈에 잘 띄어야 함 (CTA 컬러)
- PNG / PDF 선택 직관적

### 일일 제한 카운터
- "오늘 남은 변환: 2/3" 간결하게
- 제한 도달 시 부드러운 안내 (공격적 팝업 ✗)

### 랜딩 페이지
- 히어로: 한 눈에 서비스를 이해할 수 있는 비주얼 + 카피
- 사용법: 3단계 (업로드 → 변환 → 프린트)
- 난이도 소개: 상/중/하 비교 이미지
- CTA: "지금 바로 시작하기" → 업로드 영역으로 스크롤

## 🖨️ 프린트 최적화

- A4 (210mm × 297mm) 기준
- 여백: 상하좌우 15~20mm
- 선화는 흑백으로 선명하게
- 하단에 작은 watermark (ourcoloring.com)

---

## 📱 성능 가이드

- **Lighthouse 90+ 목표**
- 이미지 최적화: WebP 우선, lazy loading
- 폰트: 서브셋 + font-display: swap
- JS: React Island 패턴으로 필요한 곳만 hydrate
- Core Web Vitals: LCP < 2.5s, FID < 100ms, CLS < 0.1
